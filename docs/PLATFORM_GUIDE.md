## 1. 플랫폼 개요

### 1.1 제품 비전

MyHealth Market Lite는 "건강검진 후 90일 대사건강 개선 플랫폼"으로, 사용자의 흩어진 건강 데이터를 한곳에 모아 이해하기 쉬운 언어로 정리하고, AI 기반의 건강 행동 계획을 제공합니다.

### 1.2 핵심 사용자 가치

| 가치 | 설명 |
|------|------|
| 데이터 통합 | 검진, 진료, 웨어러블, 생활습관 데이터를 한곳에서 확인 |
| 쉬운 해석 | 검사 수치의 의미를 일반인이 이해할 수 있는 언어로 설명 |
| 행동 계획 | 90일 단위의 구체적인 건강 개선 미션 제공 |
| 진료 준비 | 의사에게 물어볼 질문과 요약 리포트 자동 생성 |
| 데이터 주권 | 본인 데이터의 동의/철회/내보내기/삭제 완전 통제 |

### 1.3 대상 사용자

- 35~65세 건강검진 이상소견 보유자
- 당뇨 전단계, 고혈압, 이상지질혈증 위험군
- 부모님 건강을 관리하는 보호자
- 기업 건강검진 후 사후관리가 필요한 임직원

## 3. 기능 상세

### 3.1 건강 대시보드

사용자의 건강 상태를 한눈에 파악할 수 있는 종합 화면입니다.

**주요 구성 요소:**

- **퀵 스탯 카드**: 오늘 걸음 수, 수면 시간, 평균 심박수, 목표 진행률
- **건강 위험 신호**: 검진 수치 중 정상 범위를 벗어난 항목을 4단계(정상/주의/경고/위험)로 표시
- **카테고리별 건강 상태**: 비만, 혈압, 당뇨, 이상지질혈증, 간기능, 신장기능별 요약
- **주간 활동량 차트**: 최근 7일간 걸음 수 바 차트
- **90일 목표 진행률**: 현재 활성 목표의 완료율과 진행 주차

**위험 신호 판정 기준:**

| 판정 | 조건 | 색상 |
|------|------|------|
| 정상 | 참조 범위 이내 | 녹색 |
| 주의 | 참조 범위 초과 (1.0~1.2배) | 노란색 |
| 경고 | 참조 범위 초과 (1.2~1.5배) | 주황색 |
| 위험 | 참조 범위 초과 (1.5배 이상) | 빨간색 |

---

### 3.2 건강검진 업로드 (OCR)

건강검진 결과지를 PDF 또는 이미지로 업로드하면 OCR로 검사 수치를 자동 추출합니다.

**지원 파일 형식:** PDF, JPEG, PNG, WebP (최대 20MB)

**처리 흐름:**
1. 파일 드래그앤드롭 또는 클릭 업로드
2. OCR 엔진이 문서에서 검사 항목과 수치를 추출
3. 참조 범위와 비교하여 자동 판정 (정상/주의/경고/위험)
4. 추출 결과를 테이블로 표시하고 사용자 확인 요청
5. 확인 후 내부 저장소에 FHIR Observation으로 저장

**추출 항목 (14개):**
BMI, 수축기/이완기 혈압, 공복혈당, HbA1c, 총콜레스테롤, LDL, HDL, 중성지방, AST, ALT, γ-GTP, 크레아티닌, eGFR

---

### 3.3 검진데이터 조회 (건강보험공단)

본인인증 후 건강보험공단에서 최근 5년간 건강검진 결과를 직접 조회하고 가져옵니다.

**전체 흐름:**

```
본인인증 수단 선택 → 인증 정보 입력 → 인증 완료 (CI/DI 발급)
    → 검진 이력 조회 (5년) → 연도별 상세 결과 확인 → 내 데이터로 가져오기
```

**본인인증 수단:**

| 수단 | 설명 |
|------|------|
| 카카오 | 카카오톡 간편인증 |
| 네이버 | 네이버 앱 간편인증 |
| PASS | 통신사 PASS 앱 인증 |
| 공동인증서 | 구 공인인증서 |

**조회 가능 항목 (15개):**
BMI, 수축기/이완기 혈압, 공복혈당, HbA1c, 총콜레스테롤, LDL, HDL, 중성지방, AST, ALT, γ-GTP, 크레아티닌, 혈색소, 백혈구 수

**인증 유효기간:** 24시간 (이후 재인증 필요)

### 3.4 일일 건강 기록

매일 측정하는 건강 수치와 복약, 증상을 기록하는 화면입니다. 4개 탭으로 구성됩니다.

#### 3.4.1 일일 기록 탭

| 측정 항목 | 입력 방식 | 단위 |
|-----------|-----------|------|
| 혈압 | 수축기/이완기 분리 입력 | mmHg |
| 혈당 | 단일 값 입력 | mg/dL |
| 체중 | 소수점 1자리 입력 | kg |

기록된 데이터는 추이 차트로 시각화됩니다.

#### 3.4.2 복약 체크 탭

- 처방받은 약물 목록 표시
- 아침/점심/저녁 시간대별 복약 체크 버튼
- 복약 완료 시 녹색 체크 표시
- 처방 기관 정보 표시

#### 3.4.3 증상 일지 탭

**선택 가능 증상 (10개):** 두통, 피로, 어지러움, 소화불량, 불면, 근육통, 관절통, 호흡곤란, 가슴답답, 부종

- 복수 증상 동시 선택 가능
- 심각도 1~5 스케일 입력
- 최근 기록 이력 표시

#### 3.4.4 알림 관리 탭

| 알림 유형 | 예시 |
|-----------|------|
| 복약 알림 | 아침 08:00, 저녁 20:00 |
| 미션 알림 | 매주 월요일 09:00 |
| 검진 알림 | 연 1회 정기 검진 |
| 기록 알림 | 월/수/금 07:30 혈압 측정 |

각 알림은 활성화/비활성화 토글로 제어합니다.

---

### 3.5 의료정보 서비스

6개 탭으로 구성된 종합 의료정보 서비스입니다.

#### 3.5.1 병원/의약품 검색

- **병원 검색**: 병원명 또는 진료과로 검색, 거리/평점 표시
- **약국 검색**: 근처 약국, 야간 운영 여부 표시
- **의약품 검색**: 약품명 또는 성분명으로 검색, 복용법 표시
- **이상소견 기반 병원 추천**: 검진 결과에서 이상 소견이 있는 카테고리에 맞는 진료과를 자동 추천하고 긴급도(즉시/2주/3개월)를 표시

#### 3.5.2 약물 상호작용 체크

현재 복용 중인 약물을 입력하면 약물 간 상호작용을 확인합니다.

| 심각도 | 설명 | 대응 |
|--------|------|------|
| 주의 (moderate) | 임상적으로 유의한 상호작용 가능 | 의사 상담 권장 |
| 참고 (low) | 경미한 상호작용 가능 | 모니터링 |

#### 3.5.3 건강 타임라인

여러 해에 걸친 검진 데이터를 시계열 차트로 표시하여 수치 변화 추이를 확인합니다.

**표시 지표:** BMI, 수축기/이완기 혈압, 공복혈당, HbA1c, 총콜레스테롤, LDL, HDL, 중성지방

#### 3.5.4 만성질환 관리 프로그램

검진 결과 기반으로 적합한 90일 관리 프로그램을 자동 추천합니다.

| 프로그램 | 대상 | 모듈 |
|----------|------|------|
| 당뇨 전단계 관리 | 공복혈당 100~125 또는 HbA1c 5.7~6.4 | 혈당 모니터링, 식이 관리, 운동 처방, 체중 관리 |
| 고혈압 관리 | 수축기 130~159 또는 이완기 85~99 | 혈압 모니터링, 나트륨 제한, 유산소 운동, 스트레스 관리 |
| 이상지질혈증 관리 | LDL 130 이상 또는 중성지방 150 이상 | 콜레스테롤 모니터링, 식이 관리, 운동 처방, 체중 관리 |

#### 3.5.5 가족/보호자 모드

부모님이나 가족의 건강을 함께 관리할 수 있는 기능입니다.

- **보호자 초대**: 이메일로 보호자 초대
- **권한 관리**: 대시보드 조회, 검진 결과 조회, 위험 알림 수신 중 선택
- **요약 뷰**: 보호자가 가족 구성원의 건강 상태 요약을 확인
- **위험 알림**: 가족의 건강 수치에 이상이 감지되면 보호자에게 알림

#### 3.5.6 데이터 내보내기

마이데이터 이동권에 따라 본인 데이터를 언제든 내보낼 수 있습니다.

| 형식 | 용도 |
|------|------|
| FHIR Bundle (JSON) | 다른 의료 시스템으로 이전, HL7 FHIR R4 표준 |
| CSV (Excel) | 스프레드시트에서 분석, Excel/Google Sheets 호환 |

### 3.7 90일 건강 목표

대사건강 개선을 위한 단계별 주간 미션 시스템입니다.

**구성:**
- 목표 제목 및 설명
- 시작일/종료일 (90일)
- 진행률 바 (완료 미션 / 전체 미션)
- 주간 미션 카드 (체크 가능)

**예시 미션:**

| 주차 | 미션 | 설명 |
|------|------|------|
| 1주차 | 매일 30분 걷기 | 점심시간 또는 퇴근 후 30분 이상 걷기 |
| 2주차 | 나트륨 섭취 줄이기 | 국물 반만 먹기, 소금 대신 레몬즙 활용 |
| 3주차 | 수면 7시간 확보 | 취침 1시간 전 스마트폰 사용 중단 |
| 4주차 | 주 3회 유산소 운동 | 빠르게 걷기, 자전거, 수영 중 택 1 |

---

### 3.8 병원 제출용 리포트

의료진에게 보여줄 건강 요약 리포트를 자동 생성합니다.

**리포트 구성:**
1. 환자 기본 정보 (이름, 생년월일, 성별)
2. 주요 검사 결과 요약 (이상 소견 중심)
3. 전체 검사 결과 테이블
4. 최근 생활 활동 데이터 (걸음 수, 수면, 활동 시간 평균)
5. 건강 관리 목표 진행 상황
6. 상담 요청 사항

**면책 조항:** 리포트는 의료적 판단이나 진단을 포함하지 않으며, 환자의 자가 기록 데이터를 기반으로 작성된 참고 자료입니다.

### 3.9 건강정보 고속도로 연동

정부 건강정보 고속도로(MyHealthWay) 테스트베드와 FHIR 기반으로 연동합니다.

**SMART on FHIR 인증 흐름:**

```
1단계: 본인인증 (카카오/네이버)
2단계: 데이터 동의 (제공동의)
3단계: OAuth2 코드 발급
4단계: 토큰 교환 (Access Token)
5단계: FHIR API 데이터 조회
```

**지원 FHIR 리소스 (12개):**

| 리소스 | 설명 |
|--------|------|
| Patient | 환자정보 |
| Organization | 의료기관정보 |
| Practitioner / PractitionerRole | 진료의정보 |
| Condition | 진단내역 |
| MedicationRequest | 약물 처방 내역 |
| Observation (Lab) | 진단검사 |
| ImagingStudy | 영상검사 |
| DiagnosticReport | 병리검사 |
| Observation (General) | 기타검사 |
| Procedure | 수술 및 처치내역 |
| AllergyIntolerance | 알레르기 및 불내성 |
| DocumentReference | 진료기록 및 기타문서 |

**API 구분:**
- 데이터 조회 API (의료데이터/공공데이터)
- 동적동의 API (제공동의/활용동의)
- 지원 API (인증, 연계지원, 플랫폼 이용, 방문기록, 기초데이터)

### 3.10 동의 관리

데이터 수집 및 활용에 대한 동의를 세분화하여 관리합니다.

**동의 카테고리:** 건강검진, 웨어러블, 생활습관, 진료기록, 투약이력, 유전체

**기능:**
- 카테고리별 동의 부여 (활용 목적 명시)
- 동의 철회 (즉시 접근 차단)
- 동의 이력 조회 (부여일, 철회일)
- 만료일 관리

**개인정보 처리 원칙:**
- 동의는 언제든지 철회 가능
- 수집 데이터는 명시된 목적으로만 활용
- 모든 접근 기록은 감사로그에 기록
- 민감정보는 AES-256 암호화 보호

## 5. AI 건강 코치

### 5.1 역할 정의

AI 코치는 의료진을 대체하지 않습니다. 사용자의 건강 데이터를 쉽게 설명하고, 생활습관 개선을 돕고, 진료 준비를 지원하는 보조 도구입니다.

### 5.2 허용/제한 기능

| 허용 | 제한 |
|------|------|
| 검진 수치의 일반적 의미 설명 | 특정 질병 확정 진단 |
| 최근 변화 추세 요약 | 약물 중단/용량 변경 지시 |
| 건강 행동 목표 제안 | 치료법 결정 |
| 식이/운동/수면/복약 습관 체크 | 검사 필요/불필요 단정 |
| 진료 때 물어볼 질문 생성 | 의료진 상담 대체 표현 |
| 응급 신호 시 병원 안내 | 유전체 기반 확률 과장 |

### 5.3 안전장치

- 모든 답변에 "참고용" 면책 문구 포함
- 고위험 키워드(흉통, 호흡곤란, 의식저하) 감지 시 즉시 응급 안내
- 진단/처방/치료 결정 문장 차단
- 사용자 건강 데이터 컨텍스트를 포함하여 개인화된 답변 제공
- LLM 실패 시 폴백 응답 (다른 모델 자동 전환 없음)

### 5.4 LLM 제공자 선택

사용자가 설정에서 원하는 LLM을 선택할 수 있습니다:
- **OpenAI**: 클라우드 기반, 높은 품질, API Key 필요
- **Google Gemini**: 클라우드 기반, 빠른 응답, API Key 필요
- **Ollama**: 로컬 실행, 데이터 외부 전송 없음, GPU 권장

## 7. 배포 및 운영

### 7.1 Docker Compose 배포

```bash
# 1. 환경변수 설정
cp .env.example .env
# .env 파일에서 OPENAI_API_KEY, POSTGRES_PASSWORD 등 설정

http://localhost
```

| redis | Redis 7 Alpine | 6379 | 캐시/세션 |
| server | Node 22 Alpine (자체 빌드) | 3001 | Express API |
| client | Nginx Alpine (Vite 빌드) | 80 | React 앱 + 프록시 |

|--------|------|

| `make status` | 상태 확인 |

| `make backup-db` | DB 백업 |
| `make clean` | 전체 정리 (볼륨 포함) |
